# Overall System Architecture

Lobster Radio is a full-stack AI radio application with:

**Frontend**: React + TypeScript + TailwindCSS - runs in the browser
**Backend**: Node.js - handles API calls to music services and AI/LLM providers
**AI Components**: Multiple AI agents work together:
  - `RadioDJAgent` - Generates intros, outros, mid-track trivia, responds to chat
  - `MemoryAgent` - Stores user preferences and chat history
  - `MusicCurator` - Generates music recommendations based on user mood/feedback

## High-Level Flow

1. User opens **[[features/radio-mode|Radio Mode]]**
2. Backend generates initial music recommendations
3. `RadioDJAgent` generates an introduction narration for the first track
4. Narration plays, then music starts
5. While music plays, we may insert **[[architecture/audio-playback|mid-track trivia]]**
6. When track ends, `RadioDJAgent` generates outro, outro plays in background while next track starts
7. User can chat with the AI DJ via **[[features/chat-producer|Chat with Producer]]** to skip tracks, request different moods, etc.

## Key Design Decisions

- **Decoupled narration playback**: Since 55bb9f2f (refactor(radio): decouple narration playback from playlist), narrations (intro/outro/trivia) use independent audio instances so skipping doesn't interrupt them. See **[[architecture/audio-playback|Audio Playback]]**.

- **LLM-first design**: All conversation content is generated by LLM - no hardcoded scripts. This gives natural variation.

- **SQLite memory**: User preferences and history are stored in SQLite for persistence across sessions (see **[[features/memory-system|Memory System]]**).
